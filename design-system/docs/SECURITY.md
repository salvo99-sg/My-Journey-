# SECURITY

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x.x | ✅ |
| < 1.0 | ❌ |

---

## Reporting a Security Vulnerability

If you discover a security vulnerability in **My Journey**, please do not disclose it publicly before it has been reviewed.

When reporting a vulnerability, include whenever possible:

- Description of the issue
- Steps to reproduce
- Potential impact
- Screenshots or logs (if applicable)
- Suggested mitigation (optional)

Reports should be treated confidentially until a fix is available.

---

## Security Principles

The project follows these principles:

- Local First architecture
- Minimum data collection
- No unnecessary third-party services
- Secure-by-default configuration
- Input validation
- Principle of least privilege
- Progressive enhancement
- Offline support without exposing sensitive information

---

## Responsible Disclosure

Security reports will be investigated, validated and resolved as quickly as reasonably possible.

Thank you for helping improve the security of My Journey.
